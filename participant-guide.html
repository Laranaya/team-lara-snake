🐍 Build a Snake Game
with GitHub Copilot

A 45-minute team challenge designed to teach you how to use Copilot's most powerful modes — in the right order.

⏱️ 45 minutes
👥 Up to 5 per team
💻 VS Code + GitHub Copilot
🎯 4 phases
Before You Start

Three modes. One challenge.

Most people use Copilot for inline code completions. Today you'll use three more powerful modes — deliberately, in order.

💬
Phase 1

Chat Mode

Ask questions, explore concepts, build understanding before writing code. Open with Cmd+Alt+I.

📋
Phase 2

Plan Mode

Switch to Plan in the Chat panel. Generate a structured implementation plan. Agree on it as a team before building.

🤖
Phases 3 & 4

Agent Mode

Switch to Agent. It reads and edits your files directly. Implement step-by-step. Use slash commands when things break.

The Challenge

4 phases. Follow the order.

Open CHALLENGE.md for full instructions. prompts.md has starter prompts for every step.

⚙️

Phase 0 – Setup

5 min
  • Clone the template repo OR download and unzip the starter files
  • Rename your folder / branch to team-[your-name]-snake
  • Open in VS Code — confirm Copilot icon is active in the status bar
  • Nominate your driver (1 person types; everyone else coaches prompts)

Chat Mode 8 min
  • Open Copilot Chat (Cmd+Alt+I Mac · Ctrl+Alt+I Win/Linux)
  • Use the starter prompt from prompts.md → Phase 1
  • Ask at least one follow-up question as a team

Phase 2 – Design

Plan Mode 8 min
  • In the Chat panel, click the mode selector → choose Plan
  • Use the starter prompt from prompts.md → Phase 2
  • Review the plan as a team — ask Copilot to adjust anything that seems wrong
Evidence required: Copy the agreed plan into evidence/phase2-plan.md

Phase 3 – Build

Agent Mode 12 min
  • Switch to Agent mode — it can now read and edit your files
  • Open snake.html — you'll see 4 TODO comments inside <script>
  • Address one TODO at a time using prompts.md → Phase 3
  • Use /explain or /fix when code is unclear or broken — required for points
Evidence required: Your best prompt + slash command usage → evidence/phase3-prompt.md
Git teams commit: feat: phase3 - base game
✨

Phase 4 – Refine

Agent Mode 8 min
  • Stay in Agent mode
  • Add at least 2 extra features (ideas in prompts.md → Phase 4)
  • Ideas: game over screen, speed ramp-up, high score, visual polish, wrap mode
Evidence required: List every feature + prompt used → evidence/phase4-features.md

Phase 3 & 4

Slash commands — your debugging superpower

You must use at least one slash command in Phase 3. It's worth 10 points — and it's genuinely useful.

/explain

Highlight a block of generated code you don't understand. Copilot will walk through it line by line.

/fix

Describe a bug you're seeing. Copilot will diagnose and fix it. Be specific — describe exactly what you observe.

Tips for a great score

🎯

Be specific. "Add keyboard controls for arrow keys that prevent reversing direction" beats "add controls" every time.

🧱

Give context. Tell Copilot what already exists in the file before asking for the next thing. Agent mode can see your file — point it in the right direction.

🔄

Iterate. A refined second prompt is almost always better than the first. Judges can tell the difference — your evidence files show the evolution.

🚫

Don't skip phases. Evidence files left blank = zero points for that phase. A team that follows the process and fills in evidence will outscore a team that skips to Agent.

👂

Everyone coaches. The driver isn't the most important role. Strong prompt coaches who refine the wording before hitting Enter make all the difference.

Phase 5

Submitting your work

Before submitting, check that your game opens in a browser and all 4 evidence files are filled in.

Option A — GitHub (recommended)

Push your repo

  2. Fill in the Google Form with your team name + repo URL
Option B — ZIP file

Upload a ZIP

  1. Zip your project folder
  2. Name it team-[name]-snake.zip
  3. Upload the ZIP file to your OneDrive
  4. Right-click the file → Share → Set access to "People in NIQ" → Copy link
